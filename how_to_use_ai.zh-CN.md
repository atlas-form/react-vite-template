# How To Use AI（如何使用 AI）

## 目标
本项目是一个 AI 标准化开发框架。
你可以直接描述需求，AI 会按照框架约定实现代码。

## 你需要告诉 AI 的信息
- 业务目标：你要新增或修改什么。
- 范围：影响哪些页面/模块/API。
- 约束：风格、性能、兼容性、安全性、截止时间。
- 验收标准：做到什么程度算完成。

## 提示词模板
向 AI 提需求时可直接使用：

```text
目标：
实现/修改 <功能>。

上下文：
项目使用 React + Vite + TypeScript。
遵循现有目录结构和代码风格。

需求：
1. ...
2. ...
3. ...

约束：
- 不能破坏现有路由。
- 保持 i18n 与主题兼容。
- 如有需要，补充或调整测试。

完成标准：
- [ ] 功能在 dev 模式可用。
- [ ] 构建通过。
- [ ] 关键文件变更清晰可审查。
```

## 框架规则
- 新增代码前优先复用已有模块。
- 可见文案统一写入 i18n 语言文件。
- 样式需要兼容当前主题变量。
- 优先小步、聚焦、边界清晰的修改。
- 交付前必须做构建或关键检查。

## 推荐流程
1. 先用一句话描述需求。
2. 明确 must-have 与 must-not-have。
3. 先让 AI 列出将要修改的文件。
4. 再让 AI 实现并验证。
5. 最后 review diff 并做最终检查。
