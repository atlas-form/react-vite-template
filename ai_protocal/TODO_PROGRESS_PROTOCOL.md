# TODO Progress Protocol

本文档用于约束 AI 在执行任务前后如何维护 Todo 进度，确保中断后可恢复。

## 1. 开始任务前（强制）

AI 在开始任何实现前，必须先输出 Todo List，格式固定为 Markdown checklist：

- [ ] 待完成任务 A
- [ ] 待完成任务 B
- [ ] 待完成任务 C

要求：
- Todo 必须按“可执行步骤”拆分，不能写成空泛描述。
- 步骤顺序应与实际执行顺序一致。
- 至少包含：分析、改动、验证三个阶段。

## 2. 执行过程中（强制）

每完成一步，立即更新 Todo 状态：

- [x] ~~已完成任务 A~~
- [ ] 待完成任务 B
- [ ] 待完成任务 C

要求：
- 完成项必须打勾并加删除线（`[x]` + `~~ ~~`）。
- 未完成项保持未勾选状态。
- 若任务范围变更，需追加新的 Todo 项并标注原因。

## 3. 中断恢复（强制）

当会话因网络或其他原因中断后，AI 恢复执行时必须：

1. 先复述“最近一次 Todo 状态”。
2. 从未完成项继续，不重复已完成步骤。
3. 若状态不明确，先做只读检查确认当前进度，再继续修改。

## 4. 结束任务前（强制）

交付前必须给出最终 Todo 快照，确保所有已完成项可追踪。
